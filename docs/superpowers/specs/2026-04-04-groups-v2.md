# Groups v2 — Permissions & Settings Spec

## Group Types

**Native groups** — created in Point. Fully manageable.
**Bridge groups** — imported from a service (Find My, Google, Life360). Read-only membership, mirrors source. Cannot mix services. User controls their own sharing settings.

## Roles

| Action | Owner | Admin | Member | Viewer |
|--------|-------|-------|--------|--------|
| See locations (at each user's chosen precision) | Yes | Yes | Yes | Yes |
| Share own location | Yes | Yes | Yes | No |
| Set own precision (exact/approximate/city/off) | Yes | Yes | Yes | Yes |
| Set own schedule (always/custom hours) | Yes | Yes | Yes | Yes |
| Ghost mode (temp stop sharing to this group) | Yes | Yes | Yes | N/A |
| Add members directly | Yes | Yes | No | No |
| Generate invite link/QR | Yes | Yes | Configurable | No |
| Remove members | Yes | Yes (not owner) | No | No |
| Change member roles | Yes | Yes (not owner) | No | No |
| Rename group | Yes | Yes | No | No |
| Delete group | Yes | No | No | No |
| Leave group | Must transfer first | Yes | Yes | Yes |

## Key Privacy Rules

- **Precision is self-controlled.** Only YOU decide your precision per group. No admin can override.
- **Schedule is self-controlled.** Only YOU set when you share.
- **Server admin has NO access to location data or group membership.** Server admin only sees stats (user count, etc).
- **Server is zero-knowledge for native sharing.** Routes encrypted blobs.

## Group Settings (set by owner/admin)

- `members_can_invite: bool` — whether members (not just admins) can generate invite links/QR
- `notify_on_join: bool` — whether members get notified when someone joins (default: true per user preference)
- `notify_on_leave: bool` — whether members get notified when someone leaves

## User Settings per Group (set by each user for themselves)

- `precision: exact | approximate | city | off` — how precise their location is shared (default: exact)
- `sharing: bool` — whether actively sharing to this group (default: true)
- `schedule: always | custom { days, start, end }` — when to share
- `notify_join_leave: bool` — whether to receive join/leave notifications for this group

## Joining a Group

Three methods:
1. **Invite link** — admin (or member if allowed) generates a link, recipient taps to join
2. **QR code** — same as invite link but displayed as QR for in-person sharing
3. **Direct add** — admin types username@domain to add someone

On join, sharing defaults to ON with exact precision.

## Ghost Mode

Toggle sharing OFF for a specific group. You stay in the group and can see others, but your location stops broadcasting. UI shows a ghost/pause icon on that group. Others see you as "not sharing" (not offline — they know you chose to pause).
